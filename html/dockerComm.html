Guía de Comandos Docker

1. docker system

Ayuda a inspeccionar y gestionar el entorno Docker, permitiendo ver el uso de disco (df), eliminar recursos no utilizados (prune), ver eventos en tiempo real (events) y mostrar informacion relacionada al sistema (info).

2. docker context

Permite navegar, crear y configurar diferentes contextos (endpoints y orquestadores) para trabajar con distintos entornos de Docker. Este comando tambien permite argumentos para mostrar los detalles default del contexto (ls), inspeccionar contextos (inspect "nombre contexto"), crear un nuevo contexto (create "nombre contexto") y cambiar entre contextos (use "nombre contexto").

3. docker pause and unpause

El comando pause congela todos los procesos activos de un contenedor, mientras que unpause los reanuda. En ambos casos se debe especificar el nombre del contenedor

4. docker rm

Elimina contenedores, volúmenes y redes específicos, permitiendo incluso forzar la eliminación de componentes en ejecución.

5. docker rmi

Se utiliza para eliminar imágenes locales mediante su ID, ayudando a mantener el nodo host limpio y eficiente.

6. docker volume

Permite la gestión completa de volúmenes (crear [create], listar [ls], inspeccionar [inspect {Nombre volumen}] y eliminar [rm {Nombre volumen}]) para el almacenamiento de datos.

7. docker search

Busca imágenes directamente en Docker Hub desde la terminal, con capacidad de aplicar filtros por estrellas u oficialidad.

8. docker push

Permite compartir imágenes locales subiéndolas al registro de Docker Hub o a un repositorio privado.

9. docker pull

Descarga imágenes desde un registro público o privado para poder utilizarlas localmente en aplicaciones contenedrizadas.

10. docker ps

Muestra la lista de contenedores en ejecución, con opciones para ver el tamaño en disco [-s] y los contenedores finalizados [-a].

11. docker tag

Crea un alias para una imagen de destino, permitiendo añadir metadatos adicionales como el número de versión.

12. docker rename

Cambia el nombre de un contenedor existente para facilitar su diferenciación según su finalidad.

sudo flatpak remote-add --if-not-exists --system flathub https://dl.flathub.org/repo/flathub.flatpakrepo

13. docker commit

Crea una nueva imagen a partir de los cambios realizados en el sistema de archivos de un contenedor.

14. docker network

Comando de gestión de red para conectar [connect], crear [create], desconectar [disconnect] o eliminar [rm] redes entre servicios y contenedores.

15. docker history

Muestra el historial de creación de una imagen específica, detallando sus capas y el tamaño de cada una.

16. docker update

Actualiza dinámicamente la configuración de un contenedor, como límites de memoria [--memory], CPUs [--cpus] o políticas de reinicio [--restart].

17. docker plugin install

Gestiona la instalación y activación de plugins para añadir nuevas funcionalidades al host de Docker. Este acepta los argumentos [create] para crear nuevos plugins, [enable] para activar plugins ya instalados, [install] para instalar nuevos plugins, [rm] para eliminar plugins y [ls] para mostrar una lista de plugins.

18. docker container

Comando general para la gestión de contenedores, permite la creación de contenedor [create], creacion de imagen tras cambios en contenedor [commit], ejecucion de comandos dentro de un contenedor ejecurandose [exec], detener un contenedor [kill], mostrar una lista de contenedores dentro de un host [ls], reiniciar un contenedor [restart], crear un contenedor a partir de una imagen y ejecutarlo [run], y por ultimo eliminar un contenedor [rm].

19. docker logs

Recupera y muestra los registros (logs) de un contenedor, información esencial para tareas de depuración.

20. docker swarm

Gestiona clústeres de Docker (enjambres), permitiendo iniciar [init], unirse [join], abandonar leave o actualizar [update] un grupo de máquinas que trabajan juntas.
